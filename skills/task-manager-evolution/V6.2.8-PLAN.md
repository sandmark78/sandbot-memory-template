### 2. 滞后领域预警

#### 实现逻辑
```python
def check_lagging_domains(progress_data: dict) -> list:
    """检测滞后领域"""
    avg_progress = progress_data['percentage']  # 平均进度
    lagging_threshold = avg_progress * 0.5  # 低于平均 50% 视为滞后
    
    lagging = []
    for domain_id, domain_data in progress_data['domains'].items():
        if domain_data['percentage'] < lagging_threshold:
            lagging.append({
                'id': domain_id,
                'name': domain_data['name'],
                'progress': domain_data['percentage'],
                'gap': avg_progress - domain_data['percentage']
            })
    
    return sorted(lagging, key=lambda x: x['progress'])
```

#### 预警级别
| 级别 | 条件 | 动作 |
|------|------|------|
| 🟢 正常 | 进度 >= 平均*0.8 | 无操作 |
| 🟡 关注 | 平均*0.5 <= 进度 < 平均*0.8 | 记录日志 |
| 🔴 滞后 | 进度 < 平均*0.5 | 发送通知 + 建议任务 |

```python
def calculate_trend(history: list, hours: int = 24) -> dict:
    """计算趋势 (过去 N 小时)"""
    cutoff = datetime.now() - timedelta(hours=hours)
    recent = [h for h in history if datetime.fromisoformat(h['timestamp']) > cutoff]
    
    if len(recent) < 2:
        return {'trend': 'insufficient_data', 'change': 0}
    
    start_points = recent[0]['total_points']
    end_points = recent[-1]['total_points']
    change = end_points - start_points
    trend = 'increasing' if change > 0 else 'decreasing' if change < 0 else 'stable'
    
    return {
        'trend': trend,
        'change': change,
        'avg_speed': change / hours,
        'prediction': end_points + (change / hours * 24)  # 24 小时预测
    }
```

### 风险 1: Cron 任务冲突
**风险**: 与其他 cron 任务时间冲突，导致资源竞争  
**缓解**: 错开执行时间 (如第 30 分钟执行)

### 风险 2: 通知骚扰
**风险**: 频繁发送通知打扰用户  
**缓解**: 设置通知频率限制 (同类通知间隔>=1 小时)

### 风险 3: 趋势数据膨胀
**风险**: trend_history.json 文件过大  
**缓解**: 自动清理 7 天前数据，保留最近 168 个点
