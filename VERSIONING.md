## 📋 版本命名规则

### SemVer 格式
```
主版本。次版本.修订号
Major.Minor.Patch

示例:
V6.1.0 - 2026-02-24: V6.1 初始版本
V6.2.0 - 2026-02-28: Timo 学习法适配
V6.2.1 - 待定：小修复

```

### 版本更新触发条件
```
主版本 (Major): 架构变更/核心理念变化
次版本 (Minor): 新功能/新方法/知识体系更新
修订号 (Patch): 小修复/文档更新/错别字修正
```

## 🔄 版本更新流程

### 1. 变更识别
```
触发条件:
- 新功能实现
- 新方法学习 (如 Timo 学习法)
- 架构优化
- 重大教训吸取
```

### 2. 版本评估
```
评估维度:
- 影响范围 (核心/边缘)
- 变更程度 (重大/中等/小)
- 向后兼容性 (兼容/不兼容)

决策:
- 核心变更 → 主版本
- 功能增加 → 次版本
- 小修小补 → 修订号
```

### 3. 文件更新
```
步骤:
1. 备份当前版本 (git commit 或手动备份)
2. 更新文件内容
3. 更新版本号 (文件头部)
4. 更新更新时间
5. 验证写入 (cat/ls 检查)
6. 记录变更日志
```

### 4. 变更日志

## 🔄 自动化版本检查

### 版本检查脚本
```bash
#!/bin/bash
# version-check.sh

echo "=== V6.1 核心文件版本检查 ==="
echo "当前版本：V6.2.0"
echo "检查时间：$(date '+%Y-%m-%d %H:%M:%S')"
echo ""

files=(
    "SOUL.md"
    "IDENTITY.md"
    "AGENTS.md"
    "MEMORY.md"
)

for file in "${files[@]}"; do
    if [ -f "$file" ]; then
        version=$(grep -oP '\*\*版本\*\*: V\K[0-9.]+' "$file" 2>/dev/null || echo "未知")
        updated=$(grep -oP '\*\*最后更新\*\*: \K.*' "$file" 2>/dev/null || echo "未知")
        echo "✅ $file: V$version (更新：$updated)"
    else
        echo "❌ $file: 文件不存在"
    fi
done
```
