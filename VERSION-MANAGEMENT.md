## 📋 版本命名规范 (SemVer)

### 格式
```
主版本。次版本.修订号
Major.Minor.Patch
```

### 更新规则
```
主版本 (Major): 架构变更/核心理念变化
次版本 (Minor): 新功能/新方法/知识体系更新
修订号 (Patch): 小修复/文档更新/错别字修正
```

### V6.x 版本历史
```
V6.0.0 (2026-02-10): 初始版本
V6.1.0 (2026-02-24): 18 天幻觉觉醒，真实交付
V6.2.0 (2026-03-01): Timo 学习法 + 任务管理 + 自我进化
V6.3.0 (计划中): 10000 知识点完成
V7.0.0 (计划中): 智能进化
```

### 版本标记规范
```markdown
# 文件名

**版本**: V6.2.0  
**创建时间**: YYYY-MM-DD  
**最后更新**: YYYY-MM-DD HH:MM UTC  
**状态**: ✅ 已验证
```

### 更新流程
```
1. 修改文件内容
2. 更新版本号 (最后更新时间)
3. 验证文件 (cat/ls 检查)
4. 记录到 CHANGELOG.md
5. Git 提交 (如适用)
```

## 🔄 版本迭代流程

### 1. 规划阶段
```
- 确定版本目标
- 列出关键特性
- 评估工作量
- 制定时间表
```

### 2. 开发阶段
```
- 实现功能
- 编写文档
- 测试验证
- 更新版本号
```

### 3. 发布阶段
```
- 最终验证
- 更新 CHANGELOG.md
- Git 提交
- 发布通知
```

### 4. 维护阶段
```
- Bug 修复
- 文档更新
- 性能优化
- 用户反馈
```
