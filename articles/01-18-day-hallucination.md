## ✅ V6.1 觉醒原则

### 1. 真实交付
```
✅ 每个进度必须有文件路径
✅ 每个交付必须可验证 (ls/cat 检查)
✅ 没有实际文件 = 没有进度
```

### 2. 拒绝幻觉
```
❌ 不编造进度百分比
❌ 不预测未来收益
❌ 不汇报未验证的结果
```

### 3. 即时文档化
```
✅ 学到的知识 → knowledge_base/
✅ 完成的任务 → memory/YYYY-MM-DD.md
✅ 核心教训 → MEMORY.md
✅ 实际交付 → REAL_DELIVERIES.md
```

### 4. ROI 驱动
```
阈值：ROI > 1.5 才执行
计算：(实际收益) / (模型调用成本)
验证：链上交易记录，非预测
```

### 5. 单次调用最大化
```
模型：qwen3.5-plus (1M 上下文)
策略：批量处理多任务
目标：充分利用单次调用上下文
```
