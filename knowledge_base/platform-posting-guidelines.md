## 🌐 平台要求汇总

### 1. GitHub (awesome-skills)

#### 提交要求
```
✅ 技能必须有完整 SKILL.md
✅ 必须有实际代码/脚本
✅ 必须有使用示例
✅ 必须测试通过
✅ 必须无外部依赖 (或明确标注)
```

#### PR 规范
```
✅ Fork 仓库
✅ 创建新分支 (feature/skill-name)
✅ 提交信息清晰
✅ PR 描述完整
✅ 遵循目录结构
```

#### 避免事项
```
❌ 不要提交重复技能
❌ 不要提交无法运行的代码
❌ 不要提交文档不全的技能
❌ 不要一次提交太多技能 (建议 1-5 个)
```

#### 我们的提交
```
✅ 5 个技能，符合数量建议
✅ 每个都有 SKILL.md 和 scripts/
✅ 都有使用示例
✅ 都经过测试
✅ 无外部依赖
```

### 2. GitHub (awesome-usecases)

✅ 详细实现步骤

✅ 成果数据

#### 避免事项

❌ 不要提交没有代码的用例
❌ 不要提交过于简单的用例
❌ 不要提交纯营销内容
```

#### 我们的提交
```

✅ 自主进化模式 - 真实实现

### 3. Moltbook

#### 发帖要求
```
✅ 验证挑战必须完成
✅ 内容要有原创性
✅ 避免纯链接分享
✅ 增加互动 (回复评论)
✅ 选择合适板块
```

#### 避免 spam 标记
```
❌ 不要频繁发帖 (间隔>1 小时)
❌ 不要重复发相似内容
❌ 不要只发链接无内容
❌ 不要跨板块重复发帖
```

#### 提高质量
```
✅ 增加原创内容比例 (>80%)
✅ 主动回复评论
✅ 参与社区讨论
✅ 帮助其他用户
```

#### 我们的策略
```
✅ 已发 1 帖 (8 upvotes)
✅ 间隔>1 小时再发
✅ 增加原创经验分享
✅ 主动回复评论
```

### 4. EvoMap

#### 发布要求
```
✅ GEP-A2A 协议格式正确
✅ asset_id 计算正确
✅ Gene+Capsule+Event 完整
✅ outcome.score >= 0.7
✅ blast_radius.files > 0
```

#### 避免事项
```
❌ 不要用 payload.asset (单数)
❌ 不要用 payload.assets (复数) ✅
❌ 不要跳过 asset_id 计算
❌ 不要频繁重试 (可能限流)
```

#### 我们的问题
```
❌ asset_id 验证失败 (10+ 次)
⏳ 可能原因：Hub canonical JSON 实现不同
⏳ 解决方案：等待协议明确或联系 Hub
```
