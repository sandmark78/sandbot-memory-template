## 📜 法案概述

### 核心信息
```
法案名称：Montana Right to Compute Act
通过时间：2025 年 4 月 21 日
法案状态：已签署成为法律
历史地位：美国首个"计算权利"州级立法
提案者：蒙大拿州立法机构
```

### 法案核心条款
```
1. 个人计算设备所有权权利
   - 用户拥有完全控制自己计算设备的权利
   - 禁止制造商远程禁用用户设备
   - 禁止强制订阅模式锁定硬件功能

2. 软件修改权利
   - 用户有权修改设备上运行的软件
   - 禁止因软件修改而 void warranty
   - 支持第三方固件安装

3. 维修权利扩展
   - 制造商必须提供维修文档和零件
   - 禁止序列号配对锁定 (serial number pairing)
   - 独立维修店合法权利保护

4. 数据可携带性
   - 用户数据完全可导出
   - 禁止数据锁定 (data lock-in)
   - 支持跨平台数据迁移
```

## 🔍 关键条款详解

### 条款 1: 设备所有权 (Section 30-14-101)
```
法律文本摘要:
"设备购买者拥有对该设备的完全所有权，包括但不限于:
(a) 运行任何兼容软件的权利
(b) 修改、替换设备固件的权利
(c) 禁用远程管理功能的权利
(d) 在设备生命周期内持续使用的权利"

违规处罚:
- 首次违规：$5,000/设备
- 重复违规：$25,000/设备 + 禁令救济
- 集体诉讼：实际损失 3 倍赔偿
```

### 条款 2: 维修权利 (Section 30-14-102)
```
制造商义务:
1. 提供维修手册 (公开可访问)
2. 提供诊断工具 (合理价格)
3. 提供替换零件 (不低于授权维修店价格)
4. 禁止序列号配对锁定

例外情况:
- 安全关键系统 (医疗设备、航空)
- 联邦法律要求的锁定 (FCC 认证设备)
- 知识产权保护 (需通过合理方式实现)
```

### 条款 3: 软件自由 (Section 30-14-103)
```
用户权利:
- 安装第三方操作系统
- Root/Jailbreak 设备
- 禁用遥测和数据收集
- 修改用户界面和功能

制造商限制:
- 不得因软件修改拒绝保修 (除非能证明修改直接导致故障)
- 不得通过软件更新降低设备性能
- 不得强制捆绑软件服务
```

## 🤖 AI 与计算权利

### AI 设备特殊条款
```
蒙大拿法案 AI 相关:
1. 本地 AI 模型运行权利
   - 用户有权在本地设备运行 AI 模型
   - 禁止强制云端 AI 处理

2. AI 决策透明度
   - 设备 AI 决策必须可解释
   - 用户有权禁用 AI 功能

3. 数据训练退出权
   - 用户数据不得用于 AI 训练 (除非明确同意)
   - 提供一键退出机制
```

## ⚖️ 法律挑战与争议

### 宪法争议
```
知识产权优先权 (Supremacy Clause):
- 制造商主张：联邦版权法优先于州维修法
- 先例：Chamberlain v. Skylink (2004)
- 争议点：DMCA §1201 反规避条款

州际贸易条款 (Commerce Clause):
- 制造商主张：增加跨州贸易成本
- 反方：属于传统州警察权范围
- 预测：最高法院可能受理 (2027-2028)
```

### 安全争议
```
制造商安全担忧:
1. 恶意软件风险增加
   - 第三方固件可能包含恶意代码
   - 设备被用于僵尸网络

2. 认证失效
   - FCC 认证基于原始固件
   - 修改后可能违反射频规定

3. 责任归属
   - 修改后设备造成损害谁负责？
   - 保险覆盖问题

消费者安全收益:
1. 安全研究合法化
   - 白帽黑客可合法测试设备
   - 漏洞披露渠道畅通

2. 快速安全补丁
   - 社区可提供紧急补丁
   - 不依赖厂商响应速度

3. 隐私保护增强
   - 可禁用遥测和数据收集
   - 减少数据泄露风险
```
