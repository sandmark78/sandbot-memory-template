## 🎯 Agent 内容优化策略

### 1. 内容协商 (Content Negotiation)
```
方法：使用 HTTP Accept 头识别 Agent
Accept: text/markdown → 判定为 Agent 请求

优势:
  - 无需额外配置文件 (如 LLMs.txt)
  - 基于现有 HTTP 标准
  - 可动态调整内容格式
```

### 2. 内容格式优化
```
人类版本 (HTML):
  - 包含导航、JavaScript、样式
  - 适合浏览器渲染
  - Token 消耗高

Agent 版本 (Markdown):
  - 纯文本内容
  - 去除无关元素
  - Token 消耗降低 60-80%
  - 解析准确性提升
```

### 3. 内容结构优化
```
关键要素:
  1. 内容顺序 - 重要信息前置
     - Agent 倾向于只读文件前 N 行
     - 核心信息放在开头

  2. 内容大小 - 精简为主
     - 去除冗余描述
     - 聚焦核心信息

  3. 节点深度 - 扁平化
     - 减少嵌套层级
     - 便于快速遍历
```

## 🔍 Agent 行为特征

### 阅读模式
```
1. 部分读取策略
   - 只读文件前 N 行/字节/字符
   - 避免上下文膨胀
   - 优先读取元数据

2. 发现 vs 告知
   - 被告知"信息存在某处" → 行为不同
   - 需要自己发现 → 可能遗漏
   - 优化：明确标注信息位置
```

### Token 优化
```
Agent 已知优化行为:
  - 选择性读取 (非全文)
  - 优先读取结构化内容
  - 避免重复内容
  - 偏好 Markdown  over HTML

优化建议:
  - 核心信息前置 (前 20% 内容)
  - 使用清晰标题层级
  - 提供内容摘要/目录
  - 去除装饰性内容
```

## 🚨 常见误区

### ❌ 错误做法
```
1. 创建独立"Agent 版"站点
   - 维护成本高
   - 内容不同步风险
   - SEO 冲突

2. 过度优化
   - 去除所有人类友好内容
   - 导致混合访问体验差
   - 忽略人类用户

3. 静态文件生成
   - 无法动态调整
   - 难以 A/B 测试
   - 更新延迟
```

### ✅ 正确做法
```
1. 动态内容协商
   - 同一 URL，不同响应
   - 实时调整
   - 易于维护

2. 渐进式优化
   - 优先优化高频页面
   - 基于数据迭代
   - 监控效果

3. 双重优化
   - 同时服务人类和 Agent
   - 内容一致，格式不同
   - 共享内容源
```
