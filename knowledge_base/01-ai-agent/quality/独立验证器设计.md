## 🎯 问题描述

当前知识获取 Cron 的问题:
```
1. 无独立验证 - 同一模型生成 + 校验

3. 自我祝贺 - 自己出题考自己
```

## 🔍 模板化检测算法

```python
def detect_template(content, threshold=0.8):
    """
    检测内容是否模板化
    
    参数:
        content: 待检测内容
        threshold: 相似度阈值 (默认 80%)
    
    返回:
        is_template: bool
        similarity_score: float
        similar_files: list
    """
    # 1. 提取内容结构
    structure = extract_structure(content)
    
    # 2. 与知识库历史内容对比
    similarities = []
    for existing_file in knowledge_base:
        existing_structure = extract_structure(existing_file)
        sim = calculate_similarity(structure, existing_structure)
        similarities.append((existing_file, sim))
    
    # 3. 找出最相似的文件
    top_similar = sorted(similarities, key=lambda x: x[1], reverse=True)[:5]
    max_similarity = top_similar[0][1] if top_similar else 0
    
    # 4. 判断是否模板化
    is_template = max_similarity > threshold
    
    return {
        'is_template': is_template,
        'similarity_score': max_similarity,
        'similar_files': top_similar
    }
```

## 🚪 质量门禁规则

### 通过标准 (全部满足)
```
✅ 来源 URL 有效 (HTTP 200)
✅ 核心主张清晰 (长度 10-200 字)
✅ 模板化检测 <80%
✅ 内容长度 >500 字
```

### 拒绝标准 (任一满足)
```
❌ 来源 URL 无效 (HTTP 404/500)
❌ 核心主张缺失或过短
❌ 模板化检测 >=80%
❌ 内容长度 <500 字
❌ 包含占位符 "[继续填充...]"
```

### 处理流程
```
1. 新知识生成 → 写入临时文件
2. 验证器运行 → 生成验证报告
3. 质量门禁决策:
   - 通过 → 移动到 knowledge_base/
   - 拒绝 → 移动到 memory/quality-failures.md
4. 每周回顾失败案例 → 优化获取策略
```

## 💰 成本估算

### 单次验证成本
```
验证模型调用：1 次
模型：qwen-max
成本：~$0.02/次
```

### 每日成本 (Cron 4 次/天)
```
4 次/天 × $0.02 = $0.08/天
$0.08 × 30 天 = $2.40/月
```

### ROI 分析
```
当前问题：60% 模板化
假设每日生成 4 文件，其中 2.4 个是垃圾
避免垃圾内容价值：难以量化，但保护知识库声誉

ROI = (避免的垃圾内容价值) / (验证成本)
    = 高 / $2.40/月
    = 高 ROI

建议：立即实施
```
