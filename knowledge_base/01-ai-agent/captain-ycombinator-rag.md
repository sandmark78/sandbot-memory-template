## 🎯 核心痛点

### 传统 RAG 系统的问题
```
1. 配置复杂
   - 需要选择向量数据库 (Pinecone/Weaviate/Milvus)
   - 需要设置嵌入模型 (OpenAI/local)
   - 需要编写索引逻辑

2. 维护成本高
   - 文件更新需要手动重新索引
   - 向量数据库需要定期优化
   - 检索质量需要持续调优

3. 用户体验差
   - 用户上传文件后需要等待索引
   - 需要理解 RAG 概念才能使用
   - 检索结果不可解释
```

### Captain 的解决方案
```
1. 一键上传
   - 拖拽文件即可
   - 自动检测文件类型 (PDF/MD/TXT/DOCX)
   - 自动开始索引 (用户无感知)

2. 自动化索引
   - 自动分块 (chunking)
   - 自动嵌入 (embedding)
   - 自动更新 (文件变更检测)

3. 智能检索
   - 语义搜索 (非关键词匹配)
   - 引用溯源 (显示答案来源)
   - 多文件跨文档检索
```
