## 🎯 核心理念

**定位**: 可教学的桌面 Agent，像人类同事一样操作你的电脑。

**核心创新**: 
- **演示一次即可**: 用户展示任务一次，Agent 提取意图 (非坐标)
- **多执行路径**: GUI、浏览器、Shell、Web、记忆、消息、调度、子 Agent 统一运行时
- **渐进式学习**: 5 层架构，从观察到主动预测

**命名来源**: 戏剧中的"替补演员"——观察主角，学习角色，必要时接替。

---

## 🏗️ 5 层渐进式架构

```
Layer 5 ┃ Proactive Autonomy        [长期方向]
         注意工作区需求，主动行动，不打扰用户
              ↑
Layer 4 ┃ Route Optimization        [部分实现]
         自动发现并升级到更快的执行路径
              ↑
Layer 3 ┃ Crystallized Memory       [部分实现]
         积累日常使用经验，固化成功路径
              ↑
Layer 2 ┃ Learn from Demonstrations [✅ 已实现]
         用户演示一次 → Agent 提取意图 → 验证 → 学习
              ↑
Layer 1 ┃ Operate Software Natively [✅ 已实现]
         操作任何人类可用的应用 —— 看、点击、输入、验证
```

### 各层依赖关系
```
每一层都依赖下一层，不能跳过。
系统通过实际使用" earns its way up" (逐步升级)。
```

## 🎬 演示教学流程

### 显式教学 (Explicit Teaching)
```
用户操作 → Agent 记录 → 提取意图 → 澄清任务 → (可选验证) → 发布技能
```

### 与宏记录器的区别
| 特性 | 传统宏记录器 | Understudy |
|------|-------------|------------|
| **记录内容** | 坐标/点击序列 | 意图 + 语义理解 |
| **适应性** | 固定坐标，UI 变化即失效 | 理解意图，适应 UI 变化 |
| **学习** | 无学习能力 | 从单次演示学习 |
| **验证** | 无验证 | 可选回放验证 |
| **发布** | 本地脚本 | 可发布 SKILL.md 技能 |

## 🛠️ 统一桌面运行时

Understudy 不是简单的 GUI 点击器，而是统一桌面运行时，混合所有执行路径：

| 路径 | 实现 | 覆盖范围 |
|------|------|----------|
| **GUI** | 13 工具 + 截图定位 + 原生输入 | 任何 macOS 桌面应用 |
| **浏览器** | Playwright 管理 + Chrome 扩展中继 | 任何网站 (含登录会话) |
| **Shell** | bash 工具，完整本地访问 | CLI 工具、脚本、文件系统 |
| **Web** | web_search + web_fetch | 实时信息检索 |
| **记忆** | 跨会话语义记忆 | 持久上下文和偏好 |
| **消息** | 8 通道适配器 | Telegram/Slack/Discord/WhatsApp 等 |
| **调度** | Cron + 一次性定时器 | 自动化重复任务 |
| **子 Agent** | 子会话并行工作 | 复杂多步委托 |

### 规划器决策
```
规划器为每个步骤决定使用哪条路径。
单个任务可能：浏览网站 → 运行 Shell 命令 → 点击原生应用 → 发送消息
全部在一个会话中完成。
```

## 👁️ GUI 定位技术

### 双模型架构
```
┌─────────────────┐      ┌─────────────────┐
│   主模型        │      │   定位模型      │
│   (决定做什么)  │      │   (决定在哪里)  │
└────────┬────────┘      └────────┬────────┘
         │                        │
         └──────────┬─────────────┘
                    │
                    ▼
         ┌─────────────────┐
         │   执行动作      │
         │   (点击/输入)   │
         └─────────────────┘
```

### 技术特性
- **HiDPI 适配**: Retina 显示屏自动适配
- **高分辨率细化**: 小目标自动高分辨率细化
- **两种定位模式**:
  - 简单预测模式
  - 多轮验证模式 (带模拟覆盖层)

### 定位基准测试
```
测试结果：30/30 目标成功定位
- 明确标签元素
- 模糊目标
- 仅图标元素
- 模糊提示
```

## 📦 技能发布系统

### 教学流程
```
1. 用户启动 Teach 模式
```

## 🔧 技术实现细节

### 坐标空间处理
- **多显示器支持**: 自动识别主/副显示器
- **坐标转换**: 逻辑坐标 ↔ 物理像素
- **稳定性**: 跨尝试反馈，自动校正

### 捕获模式
- **全屏捕获**: 整个桌面截图
- **窗口捕获**: 特定应用窗口
- **区域捕获**: 用户指定区域

### 输入模拟
- **鼠标**: 移动、点击、拖拽、滚动
- **键盘**: 文本输入、快捷键
- **触摸**: 触控板手势 (MacBook)

## 📊 与传统 RPA 对比

| 特性 | 传统 RPA (UiPath 等) | Understudy |
|------|---------------------|------------|
| **学习成本** | 高 (需工作流设计) | 低 (演示一次) |

| **维护** | 频繁 (UI 变化需更新) | 少 (意图不变即可) |

## 🎓 学习要点

### 核心知识点
1. **演示学习 (Learning from Demonstration)**: 从人类操作提取意图
