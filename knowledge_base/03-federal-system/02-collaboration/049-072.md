## 知识点 49-72: 冲突解决 (合并版)

### 49-54. 冲突类型与检测 (合并 6 个)
**定义**: 冲突是指多个 Agent 在执行任务时产生的资源/时间/优先级矛盾。

**冲突类型**:
1. **资源冲突**: 多个 Agent 争用同一资源
   - 示例：同时访问同一知识库
   - 解决：资源锁/队列等待

2. **时间冲突**: 多个任务同时到期
   - 示例：两个 P0 任务同时截止
   - 解决：优先级重评/协商延期

3. **优先级冲突**: 多个高优先级任务
   - 示例：两个 12.5 分任务
   - 解决：用户决策/业务价值重评

4. **数据冲突**: 数据不一致
   - 示例：同时修改同一知识
   - 解决：版本控制/合并策略

5. **依赖冲突**: 循环依赖
   - 示例：A 等 B，B 等 A
   - 解决：打破循环/引入中间层

6. **目标冲突**: Agent 目标不一致
   - 示例：质量 vs 速度
   - 解决：目标对齐/权衡

**检测机制**:
```python
def detect_conflict(tasks):
    conflicts = []
    for i, t1 in enumerate(tasks):
        for t2 in tasks[i+1:]:
            if t1.resource == t2.resource:
                conflicts.append(("资源", t1, t2))
            if t1.deadline == t2.deadline:
                conflicts.append(("时间", t1, t2))
            if t1.priority >= 10 and t2.priority >= 10:
                conflicts.append(("优先级", t1, t2))
    return conflicts
```

### 61-66. 死锁处理 (合并 6 个)
**死锁定义**: 多个 Agent 互相等待对方释放资源，导致都无法继续执行。

**死锁条件** (同时满足):
1. 互斥：资源一次只能一个 Agent 使用
2. 占有等待：Agent 占有资源并等待其他资源
3. 不可抢占：资源不能被强制抢占
4. 循环等待：形成循环等待链

**死锁预防**:
- 破坏互斥：资源共享化
- 破坏占有等待：一次性申请所有资源
- 破坏不可抢占：允许资源抢占
- 破坏循环等待：资源编号，按序申请

**死锁检测**:
```python
def detect_deadlock(agent_graph):
    # 检测循环等待
    visited = set()
    for agent in agent_graph:
        if has_cycle(agent, visited):
            return True
    return False
```

**死锁解除**:
1. 终止 Agent: 终止一个或多个 Agent
2. 资源抢占：抢占资源并重新分配
3. 回滚：回滚到之前状态

**V6.1 实践**:
- 预防优于解除
- 资源申请超时机制
- 定期死锁检测
