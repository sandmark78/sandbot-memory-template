## 📊 评估维度 (6 大维度)

### 1. 市场规模 ⭐⭐⭐⭐⭐
| 评分 | 标准 | 案例 |
|------|------|------|
| ⭐⭐⭐⭐⭐ | 全球市场，需求旺盛 | B2B 服务、AI 工具 |
| ⭐⭐⭐⭐ | 区域市场，稳定需求 | 内容创作、教育培训 |
| ⭐⭐⭐ | 细分市场，特定需求 | 数据分析、咨询服务 |
| ⭐⭐ | 小众市场，需求有限 | 个人工具、爱好类 |
| ⭐ | 伪需求，市场不存在 | 已验证失败的项目 |

**评估方法**:
- Google Trends 搜索趋势
- Reddit/Twitter 讨论热度
- 竞品数量和融资情况
- 目标客户群体规模

---

### 2. 竞争程度 ⭐⭐⭐⭐⭐
| 评分 | 标准 | 案例 |
|------|------|------|
| ⭐⭐⭐⭐⭐ | 蓝海市场，几乎无竞争 | OpenClaw 插件开发 |
| ⭐⭐⭐⭐ | 竞争较低，有差异化空间 | AI 客服系统 |
| ⭐⭐⭐ | 竞争中等，需差异化 | 内容创作服务 |
| ⭐⭐ | 竞争激烈，红海市场 | 电商、外卖 |
| ⭐ | 垄断市场，难以进入 | 搜索引擎、社交 |

**评估方法**:
- 直接竞品数量
- 竞品市场份额分布
- 进入壁垒高低
- 差异化可能性

---

### 3. 启动成本 💰
| 等级 | 成本范围 | 案例 |
|------|----------|------|
| **$0** | 零成本启动 | B2B 冷邮件、内容创作 |
| **$1-100** | 低成本启动 | 在线课程 (设备) |
| **$100-500** | 中等成本 | API 服务、工具开发 |
| **$500+** | 高成本启动 | AI 模型微调、DeFi |

**评估方法**:
- 硬件成本
- 软件/API 成本
- 营销成本
- 时间成本 (折算为金钱)

---

### 4. 预期收益 📈
| 等级 | 月收益范围 | 案例 |
|------|------------|------|
| **$5000+** | 高收益 | B2B 服务、企业软件 |
| **$2000-5000** | 中高收益 | 在线课程、咨询服务 |
| **$1000-2000** | 中等收益 | 内容创作、数据分析 |
| **$500-1000** | 中低收益 | 自由职业、兼职 |
| **<$500** | 低收益 | 个人项目、爱好 |

**评估方法**:
- 定价策略 (单价 × 销量)
- 客户获取成本
- 复购率/续费率
- 参考竞品收入

---

### 5. ROI (投资回报率) 🎯
**计算公式**:
```
ROI = (预期收益 - 启动成本) / 启动成本

示例:
B2B 冷邮件服务
- 启动成本：$0
- 预期收益：$5000/月
- ROI = (5000 - 0) / 0 = ∞ (理论无限，实际按时间成本算约 10.0)
```

**决策阈值**:
| ROI 范围 | 决策 | 优先级 |
|----------|------|--------|
| **ROI ≥ 8.0** | 全力执行 | P0 |
| **ROI 5.0-7.9** | 优先执行 | P1 |
| **ROI 3.0-4.9** | 允许执行 | P2 |
| **ROI 1.5-2.9** | 监控执行 | P3 |
| **ROI < 1.5** | 直接拒绝 | ❌ |

---

### 6. 实施难度 🔧
| 等级 | 难度 | 技能要求 | 时间投入 |
|------|------|----------|----------|
| **低** | 简单，可快速上手 | 基础技能 | <10 小时/周 |
| **中** | 需要学习，但可行 | 专业技能 | 10-30 小时/周 |
| **高** | 需要深入学习 | 专家技能 | >30 小时/周 |

**评估方法**:
- 技术门槛
- 学习曲线
- 时间投入
- 团队匹配度

## 📋 评估流程 (5 步法)

### Step 1: 信息收集 (30 分钟)
```
1. Google 搜索赛道关键词
2. Reddit/Twitter 查看讨论
3. 竞品分析 (网站、定价、功能)
4. 目标客户访谈 (如有条件)
```

### Step 2: 6 维度打分 (15 分钟)
```
对每个维度进行 1-5 星评分
记录评分依据和数据来源
```

### Step 3: ROI 计算 (10 分钟)
```
计算启动成本
预估预期收益
计算 ROI
```

### Step 4: 风险评估 (10 分钟)
```
识别主要风险
评估风险发生概率
制定缓解措施
```

### Step 5: 决策 (5 分钟)
```
ROI ≥ 1.5 → 允许执行
ROI ≥ 3.0 → 优先执行
ROI ≥ 8.0 → 全力执行
ROI < 1.5 → 直接拒绝
```

## ⚠️ 风险控制

### 风险类型
| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|----------|
| 市场风险 | 中 | 高 | 多赛道分散 |
| 技术风险 | 低 | 中 | 快速 MVP 验证 |
| 竞争风险 | 中 | 中 | 差异化定位 |
| 执行风险 | 高 | 高 | 每日追踪进度 |

### 止损机制
```
单一赛道连续 2 周 ROI < 1.5 → 暂停/放弃
单一赛道连续 4 周无收益 → 彻底放弃
总收益连续 2 周 <$100 → 重新评估策略
```
