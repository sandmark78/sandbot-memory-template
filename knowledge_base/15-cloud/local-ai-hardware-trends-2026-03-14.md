## 💻 消费级硬件能力 (2026)

### GPU 分级
| 级别 | 型号 | 显存 | 可运行模型 | 推理速度 |
|------|------|------|-----------|---------|
| 入门 | RTX 3060 | 12GB | 7B-13B (4-bit) | 20-40 t/s |
| 中端 | RTX 4070 | 12GB | 13B-30B (4-bit) | 30-60 t/s |
| 高端 | RTX 4090 | 24GB | 30B-70B (4-bit) | 40-80 t/s |
| 旗舰 | RTX 6000 Ada | 48GB | 70B+ (8-bit) | 60-100 t/s |

### CPU 推理 (无 GPU)
| 级别 | 型号 | RAM | 可运行模型 | 推理速度 |
|------|------|------|-----------|---------|
| 入门 | M1/M2 | 8GB | 7B (4-bit) | 5-10 t/s |
| 中端 | M2 Pro | 16GB | 13B-30B (4-bit) | 10-20 t/s |
| 高端 | M3 Max | 64GB | 30B-70B (4-bit) | 15-30 t/s |
| 旗舰 | M3 Ultra | 128GB | 70B+ (8-bit) | 20-40 t/s |

### 关键洞察
```
1. 显存是瓶颈
   - 70B 模型 (4-bit) 需要 ~35GB 显存
   - 消费级 GPU 上限 24GB (RTX 4090)
   - 解决：CPU 卸载 + 模型并行

2. 量化是关键
   - 16-bit → 4-bit 压缩 4 倍
   - 质量损失 <5% (感知不明显)
   - 速度提升 2-4x

3. Apple Silicon 优势
   - 统一内存架构
   - M3 Ultra 可达 128GB
   - 可运行 70B+ 模型 (CPU 推理)
```

## 🏗️ Sandbot 混合架构评估

### 当前架构
```
云端：bailian/qwen3.5-plus (1M 上下文)
      - 优势：大上下文、高并发
      - 劣势：按次计费、依赖网络
      - 成本：~¥500-1000/月

本地：无
      - 优势：零边际成本、隐私
      - 劣势：硬件投资、维护成本
      - 成本：一次性 ¥10,000-30,000
```

### 混合架构设计
```
┌─────────────────────────────────────┐
│ 任务分类                            │
├─────────────────────────────────────┤
│ 云端任务 (qwen3.5-plus)            │
│ - 长上下文分析 (100K+ tokens)      │
│ - 复杂推理 (多步骤)                │
│ - 知识检索 (1M+ 知识点)            │
│ - 质量要求高 (对外交付)            │
├─────────────────────────────────────┤
│ 本地任务 (70B 模型)                │
│ - 日常对话 (<10K tokens)           │
│ - 简单查询 (知识检索子集)          │
│ - 草稿生成 (初稿)                  │
│ - 内部测试 (非交付)                │
└─────────────────────────────────────┘
```

### 成本效益分析
```
当前 (纯云端):
  - 月调用：~500-1000 次
  - 单次成本：~¥0.5-2
  - 月成本：¥500-1000
  - 年成本：¥6,000-12,000

混合架构:
  - 硬件投资：¥15,000 (RTX 4090 + 64GB RAM)
  - 本地运行：70% 任务 (零边际成本)
  - 云端运行：30% 任务 (复杂/长上下文)
  - 月成本：¥150-300 (云端部分)
  - 年成本：¥1,800-3,600 + ¥15,000 (硬件)

回本周期:
  - 年节省：¥4,200-8,400
  - 回本时间：15,000 / 600 (月节省) ≈ 25 个月
  - 2 年后：纯收益 (硬件已回本)

结论：混合架构长期划算，但短期成本高。
```

### 技术挑战
```
1. 任务路由
   - 自动判断任务复杂度
   - 动态选择云端/本地
   - 失败降级 (本地→云端)

2. 知识同步
   - 本地模型知识库更新
   - 云端知识缓存到本地
   - 版本一致性

3. 质量一致性
   - 本地模型质量 < 云端
   - 用户感知差异
   - 需要透明告知

4. 运维复杂度
   - 本地模型维护
   - 硬件故障处理
   - 软件更新
```
