## 🎯 核心模式

### 版税支付机制
```
Kapwing 模式:
1. 用户使用 AI 工具生成内容
2. AI 模型训练数据来自艺术家作品
3. Kapwing 向艺术家支付版税
4. 版税来源：平台收入分成

支付公式 (推测):
版税 = (AI 生成收入 × 分成比例 × 使用权重)
```

### 支付结构
```
收入来源:
├─ 订阅费 (Pro/Enterprise)
├─ 按次付费 (Pay-per-use)
└─ 企业授权

分成比例 (推测):
├─ 艺术家：10-30%
├─ 平台：50-70%
└─ 运营/研发：20%
```

### 补偿逻辑
```
为什么支付版税？
1. 道德责任 - AI 使用艺术家数据训练
2. 法律风险 - 预防版权诉讼
3. 社区关系 - 建立创作者信任
4. 品牌差异 - "负责任的 AI"定位
```

## 💡 最佳实践

### 分成比例建议
| 角色 | 比例 | 理由 |
|------|------|------|
| 主创作者 | 50-70% | 主要工作量 |
| 贡献者 | 10-20%/人 | 辅助工作 |
| 平台 | 20-30% | 运营/推广成本 |
| 推荐者 | 5-10% | 营销激励 |

### 结算周期
```
推荐：月度结算
- 太短 (周): 管理成本高
- 太长 (季): 现金流压力
- 刚好 (月): 平衡双方需求
```

### 最低提现门槛
```
推荐：$20-50
- 太低：支付手续费占比高
- 太高：创作者现金流压力
- 刚好：平衡成本与体验
```
