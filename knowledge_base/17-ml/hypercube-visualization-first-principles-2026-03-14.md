## 📐 数学建模

### 从正方形开始 (2D)
```
建模思路：描述正方形的边 (线段集合)

水平边:
  |x| ≤ a,  |y| = a
  
  解读:
  - y 坐标固定为 -a 或 +a (上下边)
  - x 坐标在 -a 到 +a 之间变化

垂直边:
  |x| = a,  |y| ≤ a
  
  解读:
  - x 坐标固定为 -a 或 +a (左右边)
  - y 坐标在 -a 到 +a 之间变化

合并公式:
  水平线：|x| ≤ a, |y| = a
  垂直线：|x| = a, |y| ≤ a
```

### 扩展到立方体 (3D)
```
构建思路：在 XY 平面矩形基础上，添加 Z 维度

水平线 (XY 平面，Z 固定):
  |x| ≤ a, |y| = a, |z| = a
  
垂直线 (XY 平面，Z 固定):
  |x| = a, |y| ≤ a, |z| = a
  
Z 方向线 (连接前后两个面):
  |x| = a, |y| = a, |z| ≤ a

关键观察:
  - 每条规则产生 4 条线段
  - 因为约束等式的坐标有 2² = 4 种组合
  - 例如水平线：y 和 z 可以是 (-a,-a), (-a,+a), (+a,-a), (+a,+a)
```

### 推广到超立方体 (4D)
```
第四维度标记：🌀 (为避免与 w 混淆)

X 线:
  |x| ≤ a, |y| = a, |z| = a, |🌀| = a

Y 线:
  |x| = a, |y| ≤ a, |z| = a, |🌀| = a

Z 线:
  |x| = a, |y| = a, |z| ≤ a, |🌀| = a

🌀线:
  |x| = a, |y| = a, |z| = a, |🌀| ≤ a

关键特性:
  - 每条规则产生 2³ = 8 条线段
  - 总共 4 × 8 = 32 条边
  - 连接 16 个顶点
```

## 🔄 旋转数学

### 2D 旋转 (基础)
```
在 XY 平面旋转角度 α:

x_new = x_orig · cos(α) - y_orig · sin(α)
y_new = y_orig · cos(α) + x_orig · sin(α)

几何解释:
  - 旋转后 x 是邻边 (cos)
  - 旋转后 y 是对边 (sin)
  - 本质是三角函数的应用
```

### 3D 旋转 (多平面)
```
可选旋转平面:
  - XY 平面 (绕 Z 轴)
  - XZ 平面 (绕 Y 轴)
  - YZ 平面 (绕 X 轴)

复杂旋转:
  - 可以分解为多个平面旋转的序列
  - 不需要单独的"三轴旋转"公式

XY 平面旋转公式 (同 2D):
  x_new = x · cos(α) - y · sin(α)
  y_new = y · cos(α) + x · sin(α)
  z_new = z (不变)
```

### 4D 旋转 (额外自由度)
```
可选旋转平面 (6 个):
  - XY, XZ, YZ (同 3D)
  - X🌀, Y🌀, Z🌀 (新增)

为简化，聚焦 XZ 平面旋转 ("转盘"动画):

x_new = x · cos(α) + z · sin(α)
z_new = z · cos(α) - x · sin(α)
y_new = y (不变)
🌀_new = 🌀 (不变)

Z🌀 平面旋转:
z_new = z · cos(α) - 🌀 · sin(α)
🌀_new = 🌀 · cos(α) + z · sin(α)
```

## 📺 投影方法

### 问题定义
```
挑战:
  - 4D 坐标 → 2D 屏幕
  - 笛卡尔坐标轴互相正交 (90°)
  - 2D 平面只能容纳 2 个正交轴
  - 需要"不完美"的投影方案
```

### 方法 1: 斜投影 (Cavalier Projection)
```
直观方法:
  - 将 Z 轴画成 45°对角线
  - 简单易懂

公式:
  x_screen = x_model + z_model · cos(45°)
  y_screen = y_model + z_model · sin(45°)

问题:
  - 立方体看起来"拉伸"了
  - 所有边长度相同，但视觉上不自然

  原因:
  - 没有考虑透视缩短
```

### 方法 2: 柜式投影 (Cabinet Projection)
```
改进方法:
  - 缩短 Z 轴 (通常除以 2)
  - 近似真实生活中的长度收缩

公式:
  x_screen = x_model + z_model · cos(45°)/2
  y_screen = y_model + z_model · sin(45°)/2

优点:
  - 技术制图常用
  - 第一眼看起来不错

问题:
  - 旋转到某些角度 (如 70°) 会变形
  - 错误的视觉线索 (背面看起来像正面)
  - 轴的方向在现实中不可能实现
```

### 方法 3: 等轴测投影 (Isometric Projection)
```
物理上合理的方案:
  - 模型轴间距 60°
  - 现实中可以构造

公式:
  x_screen = (x_model - z_model) · cos(30°)
  y_screen = y_model + (x_model + z_model) · sin(30°)

可选优化:
  - 除以 √2 匹配旋转 45°后的长度
  - 通常不需要

优点:
  - 旋转时保持自然
  - 无变形角度
  - 游戏/工程图常用
```

### 方法 4: 4D 等轴测扩展 (有问题)
```
直观扩展:
  - 添加第四轴，间距 45°
  - x, y, z, 🌀 轴依次排列

公式:
  x_screen = -🌀_model + (x_model - z_model) · cos(45°)
  y_screen = y_model + (x_model + z_model) · sin(45°)

问题:
  - x, y, z 轴间距变成 45° (不可能角度)
  - 3D 形状会变形
  - 视觉效果混乱，难以解析
```

### 方法 5: 透视投影 (推荐)
```
3D→2D 透视:
  - 近大远小
  - 消失点效果

公式 (简化):
  x_screen = x_model / z_model
  y_screen = y_model / z_model

4D→2D 双重透视:
  - 先按 Z 除，再按🌀除
  - 产生"嵌套立方体"经典图像

公式:
  x_screen = x_model / z_model / 🌀_model
  y_screen = y_model / z_model / 🌀_model

优点:
  - 最易识别的超立方体可视化
  - 符合直觉 (类似 3D 立方体的影子)

问题:
  - Z 和🌀维度混在一起
  - 难以区分两个维度
```

### 方法 6: 鱼眼透视 (改进版)
```
思路:
  - 使用曲线"鱼眼"透视
  - 类似门镜/广角镜头效果

方法:
  - 坐标按到参考点 (相机) 的欧氏距离缩减
  - 产生弯曲的边

优点:
  - 减少边重叠
  - 更好提示内部结构
  - 可以区分"内立方体"和"外立方体"

视觉线索:
  - 内立方体的边看起来从外立方体边"后面"穿过
  - 帮助理解 4D 结构
```

### 方法 7: 混合投影 (最佳实践)
```
推荐方案:
  - X, Y, Z 用等轴测投影
  - 🌀用透视投影

公式:
  // 先做 3D 等轴测
  x_iso = (x - z) · cos(30°)
  y_iso = y + (x + z) · sin(30°)
  
  // 再做 4D 透视
  x_screen = x_iso / 🌀
  y_screen = y_iso / 🌀

优点:
  - 3D 结构保持清晰
  - 4D 维度可区分
  - YZ 平面旋转时特别稳定

效果:
  - 超立方体在 Z🌀平面旋转时
  - 看起来像"从前到后穿过自己"
  - 直观展示 4D 旋转
```

## 🎨 实现技巧

### 绘制顺序
```
深度排序:
  - 按到观察平面的欧氏距离排序
  - 先画远的，后画近的
  - 正确处理遮挡关系

实现:
  - 计算每条边的中点深度
  - 按深度排序边
  - 依次绘制
```

### 动画建议
```
推荐旋转:
  1. XZ 平面 (转盘式)
     - 最直观
     - 类似 3D 物体旋转
  
  2. Z🌀平面 (4D 特有)
     - 展示 4D 特性
     - "穿过自己"的效果
  
  3. 复合旋转
     - 同时旋转多个平面
     - 更炫但更难理解
```

### 视觉增强
```
着色策略:
  - 按深度调整亮度
  - 远的边更暗
  - 增强立体感

交互建议:
  - 鼠标控制旋转角度
  - 滑块调整投影参数
  - 切换不同投影方法
```

## 🧠 认知洞察

### 为什么超立方体这么难理解？
```
1. 缺乏日常经验
   - 没有 4D 物体的触觉经验
   - 只能依赖数学抽象

2. 投影信息丢失
   - 4D→2D 丢失 2 个维度
   - 不同投影强调不同特性
   - 没有"正确"的可视化

3. 旋转反直觉
   - 4D 旋转平面有 6 个
   - 3D 生物只能想象 3 个
   - X🌀, Y🌀, Z🌀平面旋转难以可视化
```

### 类比理解法
```
3D→2D 类比:
  - 想象 3D 立方体在 2D 墙上的影子
  - 影子会变形，但保留结构信息
  - 超立方体→3D 也是类似关系

嵌套立方体解释:
  - 小立方体 = 超立方体的"近端"
  - 大立方体 = 超立方体的"远端"
  - 连接线 = 4D 方向的边
```
