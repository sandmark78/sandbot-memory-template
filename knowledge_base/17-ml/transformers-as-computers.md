## 技术背景

### 传统方法
```
LLM + 外部解释器：
  1. LLM 生成代码
  2. 外部解释器执行
  3. 结果返回 LLM

问题：
  - 多次 API 调用 (昂贵)
  - 延迟高
  - 上下文切换
  - 错误处理复杂
```

### 新方法：内部执行
```
Transformer 直接执行：
  1. 程序编码为输入
  2. Transformer 前向传播 = 执行步骤
  3. 输出 = 程序结果

优势：
  - 单次前向传播
  - 无外部依赖
  - 可微分 (可梯度优化)
  - 并行执行
```

## 理论基础

### Transformer 的计算能力
```
图灵完备性：
  - Transformer 理论上可模拟任何计算
  - 需要足够层数/注意力头
  - 位置编码提供"时间"维度

实际限制：
  - 上下文窗口有限
  - 精度有限 (浮点数)
  - 训练数据分布
```

### 程序表示
```
编码方式：
  - 程序 = token 序列
  - 状态 = 隐藏表示
  - 执行 = 状态转换

示例：
  输入："x = 1; y = 2; z = x + y"
  隐藏状态：[x:1, y:2, z:?]
  输出："z = 3"
```

## 性能分析

### 推理速度
```
传统方法：
  - 生成代码：O(n) tokens
  - 执行：O(m) 操作
  - 总延迟：生成 + 执行 + 通信

内部执行：
  - 单次前向传播：O(1)
  - 并行计算
  - 指数级加速

关键：
  "指数级更快的推理"
```

### 可扩展性
```
限制因素：
  - 模型大小 (参数数量)
  - 上下文窗口 (程序长度)
  - 数值精度 (计算准确性)

优化方向：
  - 专用架构 (更适合计算)
  - 混合精度 (关键计算高精度)
  - 缓存机制 (重用中间状态)
```

## 与其他工作的联系

### Neural Turing Machines
```
NTM (2014):
  - 神经网络 + 外部内存
  - 可学习读写操作
  - 图灵完备

对比：
  - NTM: 显式内存 + 控制器
  - Transformer: 隐式内存 (注意力)
  - 本工作：程序作为输入，执行作为前向传播
```

### Differentiable Computers
```
可微分解释器：
  - 程序可微分
  - 梯度反向传播到程序
  - 可学习程序

对比：
  - 传统：解释器固定，学习输入
  - 本工作：解释器即模型，端到端
```

### Chain of Thought
```
CoT (2022):
  - LLM 逐步推理
  - 中间步骤显式
  - 提高复杂任务准确性

对比：
  - CoT: 生成中间步骤 (token)
  - 本工作：中间状态在隐藏层
  - 本工作更高效 (无额外 token)
```

## 局限性

### 当前限制
```
1. 程序复杂度
   - 只能处理简单程序
   - 复杂程序需要更大模型

2. 数值精度
   - 浮点运算不精确
   - 不适合高精度计算

3. 可解释性
   - 隐藏状态难以理解
   - 调试困难

4. 训练数据
   - 需要大量程序 - 执行对
   - 数据生成成本高
```
