### 1-8. 记忆层次结构
**定义**: 记忆系统按照时间维度和重要性分为不同层次。

**层次划分**:

1. **短期记忆 (Short-term Memory)**
   - 保持时间：秒到分钟
   - 容量有限：7±2 个信息块
   - V6.1 应用：当前对话上下文
   - 优化：及时转化为长期记忆

2. **工作记忆 (Working Memory)**
   - 保持时间：分钟到小时
   - 用于当前任务处理
   - V6.1 应用：当前任务上下文
   - 优化：任务完成后归档

3. **长期记忆 (Long-term Memory)**
   - 保持时间：天到永久
   - 容量理论上无限
   - V6.1 应用：MEMORY.md, knowledge_base/
   - 优化：定期整理和压缩

4. **情景记忆 (Episodic Memory)**
   - 记录具体事件和场景
   - 有时间标签
   - V6.1 应用：memory/YYYY-MM-DD.md
   - 优化：按时间线组织

5. **语义记忆 (Semantic Memory)**
   - 记录概念和知识
   - 无时间标签
   - V6.1 应用：knowledge_base/
   - 优化：按主题组织

6. **程序记忆 (Procedural Memory)**
   - 记录技能和流程
   - 自动化执行
   - V6.1 应用：scripts/*.py
   - 优化：模块化和复用

7. **元记忆 (Metamemory)**
   - 关于记忆的记忆
   - 记忆的可访问性判断
   - V6.1 应用：memory/index.md
   - 优化：记忆索引和检索

8. **集体记忆 (Collective Memory)**
   - 多个 Agent 共享的记忆
   - 知识共享和同步
   - V6.1 应用：7 子 Agent 共享知识库
   - 优化：知识同步机制

**V6.1 记忆层次实践**:
```
✅ 短期记忆：对话上下文 (1M tokens)
✅ 工作记忆：任务上下文 (memory/tasks.md)
✅ 长期记忆：MEMORY.md + knowledge_base/
✅ 情景记忆：memory/YYYY-MM-DD.md
✅ 语义记忆：knowledge_base/领域/
✅ 程序记忆：scripts/*.py
⏳ 元记忆：memory/index.md (待完善)
⏳ 集体记忆：7 子 Agent 知识同步 (待实现)
```
