## ADR-002: 模型路由层实现

**日期**: 2026-02-26  
**状态**: ✅ 已采纳  
**借鉴**: ClawRouter 成本优化理念

### 背景
```
问题：模型调用成本较高，所有任务都用同一模型
需求：根据任务复杂度选择合适模型
约束：不替换现有 Bailian 配置
```

### 决策
```
✅ 添加智能路由层 (scripts/model_router.py)
✅ 根据任务复杂度选择模型
   - SIMPLE → qwen3.5-turbo ($0.5/M)
   - MEDIUM → qwen3.5-plus ($2.0/M)
   - COMPLEX → qwen-max ($10.0/M)
✅ 成本追踪和统计

❌ 不替换 Bailian 配置
❌ 不修改 openclaw.json
❌ 不影响现有功能
```

### 后果
```
正面:
+ 预计节省 70-90% 模型成本
+ 自动选择合适模型
+ 成本透明可追踪

负面:
- 增加路由层复杂度
- 需要维护模型配置
- 可能误判任务复杂度
```

### 文件
- `scripts/model_router.py` ✅

## ADR-004: 拒绝容器隔离

**日期**: 2026-02-26  
**状态**: ✅ 已拒绝  
**参考**: NanoClaw 设计

### 背景
```
问题：NanoClaw 使用容器隔离，更安全
需求：评估是否采用
约束：保持 7 子 Agent 协作能力
```

### 决策
```
❌ 拒绝容器隔离

理由:
- 不适合多 Agent 协作场景
- 增加部署复杂度
- 与现有架构不兼容

我们的选择:
✅ 保持进程级隔离
✅ 专注协作能力
✅ 渐进式改进
```

### 后果
```
正面:
+ 保持现有协作能力
+ 部署简单
+ 架构一致

负面:
- 安全性不如容器
- 需要其他方式保障安全
```
