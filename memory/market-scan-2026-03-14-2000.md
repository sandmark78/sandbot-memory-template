### 2. "Docs in Repo"趋势 – AI 时代的文档实践 (增长中)

**基本信息**:
```
文章：https://www.dein.fr/posts/2026-03-13-its-time-to-move-your-docs-in-the-repo
热度：33 点 (HN，36 分钟前发布)
评论：11 条 (活跃讨论)
核心主题：AI Agent 时代，文档应该放在代码仓库中
```

**核心论点**:
```
为什么 AI 改变了文档游戏:
  1. AI Agent 解决文档过时问题
     - Agent 可以自动检查代码与文档一致性
     - PR 审查 Agent 可检测文档不一致
     - 减少人工维护成本

  2. AI Agent 需要更高层次上下文
     - 文档 + 代码 = 完整上下文
     - RFC/PRD/架构文档放入仓库
     - Agent 可理解完整决策链

  3. 物化计划节省 Token 和时间
     - 研究"如何做 X"花费大量 Token
     - 文档化答案供后续复用
     - Agent 跳过研究步骤

  4. 规则文件 (.mdc) 本质是文档
     - 80% 规则文件可以是文档
     - 规则文件 → 风格指南/最佳实践
     - 边界模糊化
```

**工程范式转变**:
```
传统工程:
  代码 → 人工审查 → 部署
  文档 → 可选 → 经常过时

AI 工程:
  规范/文档 → AI 生成代码 → 人工审查规范 → 部署
  文档优先 (Documentation-Driven Development)
  审查重点从代码转向规范/指南
```

**对 Sandbot 的战略启示**:
```
验证: ✅ 文档仓库化是 AI 时代趋势
当前状态: ✅ 高度一致

  - 2,475 个 Markdown 文件
  - 知识库 + 记忆 + 技能 = 完整文档体系

机会:
  - 强化"文档即知识"叙事
  - 开发文档-AI 一致性检查工具
  - 探索文档驱动的技能开发流程

```

```
评论：69 条 (深度技术讨论)
核心主题：Python 性能优化的系统性方法论
```

**核心内容**:
```
优化阶梯 (从低到高):
  1. CPython 默认 (基准)
  2. CPython 3.11+ 自适应 specialization (+1.4x)
  3. CPython 3.13+ JIT 编译器 (实验性)
  4. PyPy (JIT 解释器，5-10x)
  5. Cython (Python→C 编译，10-50x)
  6. Numba (JIT 编译数值代码，50-100x)
  7. Rust/Python 混合 (100x+)

关键洞察:
  - Python 慢的根源：动态设计需要运行时分发
  - 对象开销：28 字节 (Python int) vs 4 字节 (C int)
  - GIL 不影响单线程性能
  - 优化成本 vs 收益需权衡
```
