### 2. What CI Looks Like at a 100-Person Team (PostHog) (28 分，8 评论) NEW!
**来源**: https://www.mendral.com/blog/ci-at-scale

**核心数据** (一周统计):
```
PostHog CI 规模:
- 575,894 CI 任务 / 94,574 工作流运行
- 11.8 亿行日志 (76.6 GiB)
- 3340 万次测试执行 / 22,477 个独特测试
- 3.6 年计算时间 (单周)
- 每天 65 次 main 合并 / 105 个 PR 测试
- 98 个活跃贡献者
- 每次 main 提交触发 221 个并行任务

关键问题:
- 测试通过率 99.98% (优秀)
- 但 14% 计算资源用于失败/取消
- 3.5% 任务是重跑
- "flake test"在规模下成为必然
```

**Mendral AI Agent 解决方案**:
```
核心功能:
1. 日志摄取：每秒处理 10 亿 + 日志行，可实时搜索
2. Flake 检测：关联错误与代码变更，追溯根源
3. 自动修复：识别 flake 后自动开 PR 修复/隔离
4. Slack 集成：精准通知责任人 (非群发)
5. 持续分析：构建仓库健康度画像

技术亮点:
- 日志摄取是瓶颈 (非 AI 诊断)
- Flake 几乎都有确定性根因 (时序/共享状态/基础设施)
- 路由问题被低估 (知道通知谁 ≈ 知道什么失败)
- 公开仓库保持透明 (所有 PR 可见)
```
