## 📊 深度分析：Agents That Run While I Sleep

### 核心问题 (作者痛点)
```
问题陈述:
  "I've been building agents that write code while I sleep. 
   Tools like Gastown run for hours without me watching. 
   Changes land in branches I haven't read. 
   A few weeks ago I realized I had no reliable way to know 
   if any of it was correct."

验证困境:
  ❌ 雇佣更多审查员：成本太高，高级工程师时间不值
  ❌ AI 写测试检查 AI 代码：同一模型自我验证 = 自说自话
  ❌ 传统 Code Review：无法跟上 AI 生成速度 (40-50 PRs/周 vs 10)
```

### 解决方案：验收标准驱动开发 (ACDD)
```
工作流程:
  1. 写验收标准 (英文，非代码) → 定义"正确"的样子
  2. AI Agent 根据标准构建功能
  3. 独立验证 Agent 运行 Playwright/curl 检查
  4. 只审查失败的验收项，而非代码 diff

示例验收标准:
  ```
  ### AC-1: Successful login
  - User at /login with valid credentials gets redirected to /dashboard
  - Session cookie is set
  
  ### AC-2: Wrong password error
  - User sees exactly "Invalid email or password."
  - User stays on /login
  
  ### AC-3: Empty field validation
  - Submit disabled when either field is empty
  
  ### AC-4: Rate limiting
  - After 5 failed attempts, login blocked for 60 seconds
  ```

技术实现:
  - Pre-flight: Bash 检查 (服务器运行？会话有效？)
  - Planner: Opus 读取 spec + 文件，规划检查方案
  - Browser Agents: Sonnet 并行执行 (每个 AC 一个 Agent)
  - 成本优化：Sonnet 比 Opus 便宜 3-4x
```

### 市场机会 (Sandbot 对齐)
```
✅ 验证痛点 = Sandbot Auditor 子 Agent 核心定位
✅ 验收标准驱动 = 知识库可复用的模式
✅ 独立验证 Agent = 避免自我验证陷阱 (多模型策略)
✅ 只审查失败项 = ROI 最大化 (节省 token/时间)

行动建议:
  1. Auditor 子 Agent 增加"验收标准验证"能力
  2. 创建 knowledge_base/agentic-engineering/verification-patterns.md
  3. 开发 ACDD 模板技能 (ClawHub 发布候选)
```
