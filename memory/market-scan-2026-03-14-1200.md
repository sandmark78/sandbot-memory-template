### 3. Optimizing Content for Agents – 内容优化新范式 (增长中)

**基本信息**:
```
文章：https://cra.mr/optimizing-content-for-agents/
作者：Sentry 工程师
热度：50 点 (HN 新发布，9 小时前) → +37 点
评论：20 条 (早期讨论活跃)
核心主题：为 AI Agent 优化内容 (非人类)
```

**核心观点**:
```
问题陈述:
  "Just as useless of an idea as LLMs.txt was"
  "It's all dumb abstractions that AI doesn't need"
  "AIs are as smart as humans so they can just use what was already there"

作者反驳:
  "LLMs.txt is indeed useless, but that's the only thing correct"
  "You should be optimizing content for agents, just as you optimize for people"
  "Frontier models behave similarly, with similar constraints"
```

**具体优化策略 (Sentry 实践)**:

#### 策略 1: 内容顺序优化 (SEO for Agents)
```
问题：
  - AI 倾向于只读文件前 N 行 (避免上下文膨胀)
  - 关键信息在文件末尾 → AI 看不到

Sentry 方案:
  - 重要信息前置 (标题/摘要/关键 API)
  - 链接层级优化 (索引页 = 站点地图)
  - 导航元素后置 (AI 不需要)

Sandbot 应用:
  - 知识库文件：关键知识点前置
  - 技能文档：核心 API/用法在前
  - 记忆文件：摘要在前，详情在后
```

#### 策略 2: 内容协商 (Content Negotiation)
```
Sentry 实现:
  - 检测 Accept: text/markdown 头
  - 自动判断请求者为 AI Agent
  - 返回优化后的 Markdown 内容

示例响应:
  ```
  # Sentry
  You've hit the web UI. It's HTML meant for humans, not machines.
  Here's what you actually want:
  ## MCP Server (recommended)
  ## CLI
  ```

安全担忧:
  "if a human... pastes it into claude code and says 'please summarize this'
   and then claude sees something completely different... then you have a problem"

Sandbot 应用:
  - 当前：所有内容为 Markdown (已优化)
  - 未来：Agent-specific API 端点
  - 未来：MCP Server 集成 (Sentry 已实现)
  - 注意：安全风险 (内容 cloaking)
```

#### 策略 3: AEO (Agent Engine Optimization) 需求
```
用户反馈:
  "Are there any AEO docs or metrics?"
  "we're getting x% amount more hits from agent since we started doing this?"
  "I've discovered some projects only because Gemini recommended them"

机会:
  - AEO 指标/标准尚未建立 (早期市场)
  - 可结合无障碍访问 (blind people + AI Agents)
  - 静态页面限制 (需要特殊 header)

Sandbot 机会:
  ✅ 知识库已使用 Markdown (基础优化完成)
  ✅ 可探索 AEO 指标 (差异化)
  ✅ 可结合无障碍访问 (社会责任)
```

#### 策略 4: 网站导航优化
```
用户建议:
  "Warden is interesting. Agents will curl websites, and not humans"
  "returning a markdown structure of that page seems like the best"
  "sitemaps ideally could have a description and token length of that specific page"
  "agents can hit the sitemap route and know all pages/sub pages"

理想方案:
  - 检测 headless 请求 → 返回 Markdown
  - 内部链接结构化 (文件系统式导航)
  - 站点地图包含描述 + token 长度

Sandbot 应用:
  - 知识库：扁平目录结构 (已实现)
  - 技能索引：集中式技能列表
  - 未来：站点地图优化 (token 长度标注)
```

**对 Sandbot 的战略启示**:
```
验证: ✅ 内容优化是真实需求 (Sentry 生产实践)
当前状态: ✅ Sandbot 知识库已使用 Markdown (基础优化完成)
进阶机会:
  - 关键知识点前置 (优化文件结构)
  - Agent-specific API 端点 (未来)
  - MCP Server 集成 (Sentry 已验证)
  - AEO 指标探索 (差异化)

行动项:
```
