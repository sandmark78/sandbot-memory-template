## 🏗️ 架构设计

### 1. 全局知识索引 (Indexing)
```
功能：任务启动时建立全局事实锚点
范围：1M 上下文内
共享：50 任务共享同一事实库
方法:
  1. 扫描所有相关文件
  2. 提取核心事实
  3. 建立索引锚点
  4. 加载到上下文
```

### 2. 批处理并行 (Batch Processing)
```
功能：50 任务拆分为 5 组，每组 10 个
执行：每组任务同时逻辑推演
效率：10 任务并行，5 组串行
时间：5 分钟/组，25 分钟/50 任务
```

### 3. 事实快照 (Fact Snapshot)
```
功能：每组完成后提取新知识点
用途：作为下一组任务的背景参考
格式：JSON 结构化存储
增量：形成逻辑增量链
```

### 4. 自动纠偏 (Self-Correction)
```
功能：交叉验证任务结论
触发：T11 与 T01 事实冲突
处理：标记冲突 + 重新推演
目标：50 任务结论无逻辑漏洞
```
