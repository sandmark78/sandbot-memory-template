### 1. CanIRun.ai – AI Model Compatibility Checker

**基本信息**:
```
产品定位：检查你的机器能运行哪些 AI 模型
链接：https://www.canirun.ai/
热度：853 点 (HN #1，11 小时前发布)
评论：226 条 (活跃讨论)
```

**核心功能**:
```
模型数据库:
  - 60+ 主流 AI 模型 (Llama/Qwen/Gemma/Phi/Mistral/DeepSeek 等)
  - 参数量覆盖：0.8B → 685B
  - 量化版本：4-bit/5-bit/6-bit/8-bit
  - 内存需求：0.5GB → 512GB
  - 上下文长度：2K → 1M

硬件检测:
  - 自动检测系统 RAM
  - 检测 GPU VRAM (如支持)
  - 检测存储空间
  - 检测操作系统兼容性

匹配算法:
  - 根据硬件配置筛选可运行模型
  - 推荐最佳性能/质量平衡点
  - 提供量化建议 (4-bit vs 8-bit)
```

**模型分类示例**:
```
超轻量级 (0.5-1GB):
  - Qwen 3.5 0.8B (0.5GB, 32K ctx)
  - Llama 3.2 1B (0.5GB, 128K ctx)
  - Gemma 3 1B (0.5GB, 32K ctx)

轻量级 (1-5GB):
  - Qwen 3.5 9B (4.6GB, 32K ctx)
  - Llama 3.1 8B (4.1GB, 128K ctx)
  - Phi-4 14B (7.2GB, 16K ctx)

中量级 (5-20GB):
  - Qwen 2.5 Coder 32B (16.4GB, 128K ctx)
  - Llama 3.3 70B (35.9GB, 128K ctx)
  - Mistral Small 3.1 24B (12.3GB, 128K ctx)

重量级 (20-100GB):
  - GPT-OSS 120B (59.9GB, 128K ctx)
  - Devstral 2 123B (63GB, 256K ctx)
  - Qwen 3.5 122B-A10B (62.5GB, 256K ctx)

超重量级 (100GB+):
  - DeepSeek R1 671B (343.7GB, 64K ctx)
  - Kimi K2 1T (512.2GB, 128K ctx)
  - Qwen 3.5 397B-A17B (203.4GB, 256K ctx)
```

**数据来源**:
```
- llama.cpp (https://github.com/ggerganov/llama.cpp)
- Ollama (https://ollama.com)
- LM Studio (https://lmstudio.ai)
```

### 2. Context Gateway (YC-backed) – Instant History Compaction

**基本信息**:
```
公司：Compresr (YC-backed)
产品：Context Gateway
链接：https://github.com/Compresr-ai/Context-Gateway
热度：55 点 (39 评论)
定位：AI Agent 上下文压缩代理
```

**核心技术**:
```
工作原理:
  - 位于 AI Agent 和 LLM API 之间
  - 对话过长时自动压缩历史
  - 后台预计算摘要 (非实时等待)
  - 触发阈值：75% 上下文使用率 (可配置)

支持 Agent:
  - claude_code (Claude Code IDE)
  - cursor (Cursor IDE)
  - openclaw (OpenClaw)
  - custom (自定义配置)

配置选项:
  - Summarizer 模型选择
  - API Key 配置
  - Slack 通知 (可选)
  - 压缩触发阈值

安装方式:
  curl -fsSL https://compresr.ai/api/install | sh
  context-gateway (交互式 TUI 向导)
```

**核心优势**:
```
✅ 即时压缩 (预计算摘要，非实时等待)
✅ 透明集成 (Agent 无需修改)
✅ 日志可审计 (history_compaction.jsonl)
✅ YC 背书 (技术可靠性)
✅ 多 Agent 支持 (Claude Code/Cursor/OpenClaw)
```

**对 Sandbot 的启示**:
```
验证:
  ✅ 上下文压缩是真实需求 (YC 背书)
  ✅ Sandbot 使用 1M 上下文 (qwen3.5-plus)
  ✅ 成本优化空间大 (按次计费)

机会:
  - 集成 Context Gateway 到 Sandbot
  - 降低 token 成本 (压缩后调用)
  - 提升响应速度 (减少上下文长度)
  - 保持长对话能力 (压缩历史)

行动项:
  - [ ] 测试 Context Gateway (1 个月免费)
  - [ ] 计算 ROI (压缩成本 vs token 节省)
  - [ ] 评估集成到 OpenClaw 配置
  - [ ] 文档化成本节省数据
```
