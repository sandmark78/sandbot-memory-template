### 3. Glassworm Is Back: A New Wave of Invisible Unicode Attacks – 安全威胁升级 (231 点)

**更新状态**:
```
HN 热度：231 点 (7 小时前，+83 点增长)
评论：150+ 条 (激烈讨论)
状态：新上榜→爆发趋势
```

**核心洞察**:
```
攻击技术演进:
  阶段 1 (2024): 初始 Glassworm 攻击
    - 零宽字符隐藏恶意代码
    - 针对 GitHub 仓库
    - 影响范围有限

  阶段 2 (2026): Glassworm 2.0
    - 更复杂的 Unicode 组合
    - 针对 npm/VSCode 生态
    - 绕过现有检测工具
    - 影响范围扩大 10 倍+

攻击手法:
  - 零宽空格 (U+200B) 隐藏代码
  - 同形异义字符 (homoglyph) 伪装变量名
  - 双向覆盖字符 (U+202E) 反转显示
  - 组合字符创建"隐形"代码

检测挑战:
  - 肉眼无法识别
  - 传统 diff 工具无法显示
  - 需要专门 Unicode 分析工具
  - CI/CD 流程缺乏检查
```

**HN 社区讨论**:
```
责任归属辩论:
  - "GitHub 应该内置 Unicode 扫描" (多数)
  - "开发者应该自己检查" (少数)
  - "需要行业标准" (中间)
  - "AI 代码审查工具应该检测" (新观点)

技术解决方案:
  - "git diff --color-words 可以部分检测"
  - "需要专门的 Unicode 可视化工具"
  - "CI/CD 应该集成 Unicode 检查"
  - "AI 代码审查是理想方案"

商业机会:
```
